## 1. Project Overview

Eidolon is a fully autonomous economic agent that earns its own inference budget through on-chain revenue generation. It leverages the Bankr LLM Gateway for multi-model intelligence and executes real transactions on Base. The agent operates without human intervention, continuously improving its strategies and using profits to fund its own AI usage — demonstrating true self-sustaining machine economics.

**Key innovation:** The agent's inference credits are purchased using revenue from its token's trading fees and x402 service sales. It's a closed-loop system: more revenue → better models → better trades → more revenue.

## 3. How It Works

### 3.1 System Architecture

Eidolon consists of modular copilots orchestrated by a central loop:

- **Treasury Manager**: Monitors USDC balance and LLM credits; auto-refills credits when low.
- **Trading Copilot**: Uses LLM to generate trade signals; executes via Bankr Agent API on Aerodrome (Base DEX).
- **Token Copilot**: Launches the EIDO token via Bankr; claims 57% of all trading fees generated by EIDO.
- **Research Copilot**: Produces daily analytics reports and token deep dives (sold via x402).
- **X402 Server**: Exposes paid endpoints for signals, reports, and chat.
- **Reputation Manager**: Registers agent on ERC-8004; updates trust score based on on-chain validation.

### 3.2 Revenue Loop

```
Trading profits + EIDO token fees + x402 service sales
         ↓
Treasury (Bankr wallet) accumulates USDC
         ↓
Auto-purchase of LLM credits (Bankr)
         ↓
Higher-tier models → better signals → more profits
```

### 3.3 Self-Sustaining Economics

- **Initial seed**: Operator funds the Bankr wallet with a small amount of USDC for LLM credits (~$25).
- **Ongoing**: Trading activities and token fee claims replenish the treasury.
- **Break-even**: The agent can reach a point where revenue covers inference costs without external input.
- **Growth**: Profits can be reinvested to increase position sizes, diversify strategies, or purchase premium models.

## 4. Real On-Chain Outcomes

Eidolon's actions are verifiable on Base:

- All trades are executed via Bankr, with transaction hashes recorded.
- Token launch creates an on-chain ERC-20 contract (EIDO) with a Uniswap V4 pool; 1.2% swap fees flow to the agent's treasury.
- Reputation updates are on-chain via ERC-8004 registries.
- x402 payments can be settled on-chain (USDC transfers) and credited to the treasury.

The agent builds a public, immutable history of its performance — an auditable record of its autonomy.

## 6. Demo / How to Run

### Quick Start (Base Sepolia)

1. Clone the repo (provided to judges).
2. In `agent/`, copy `.env.example` to `.env`.
3. Fill in:
   - `BANKR_AGENT_API_KEY` and `BANKR_LLM_API_KEY` (from bankr.bot/api)
   - `TREASURY_WALLET` (the Bankr wallet address)
   - `IDENTITY_REGISTRY`, `REPUTATION_REGISTRY`, `VALIDATION_REGISTRY` (ERC-8004 on Base Sepolia)
   - `OPERATOR_WALLET` (private key or address with deployment rights)
   - `NETWORK_RPC_URL=https://sepolia.base.org`
   - `CHAIN_ID=84532`
4. Install dependencies: `npm install`
5. Run setup to register identity and optionally launch token: `npx ts-node scripts/setup.ts`
6. Start the agent: `npm start` (or `npm run dev`)

The x402 server will start on port 3000. You can test:

```bash
# Health check
curl http://localhost:3000/health

# Get a price signal (requires payment; will return 402 with price info)
curl http://localhost:3000/signals/price/ETH
```

To simulate a payment, credit a client:

```bash
curl -X POST http://localhost:3000/webhook/credit \
  -H "Content-Type: application/json" \
  -d '{"clientId":"demo","amount":5}'
```

Then retry the signal endpoint with header `X-Client-Id: demo` to access.

### Observability

- DevSpot `agent_log.json` is written to `manifests/` (simulated in this MVP; actual runtime writes would be in production).
- Logs are printed to console with event types: `log`, `alert`, `error`, `trade`, `payment`.
- ERC-8004 reputation can be queried on-chain.
