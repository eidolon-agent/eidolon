📡 Overview

Eidolon is an autonomous self-sustaining economic agent built on the Bankr LLM Gateway. It provides x402 payment infrastructure, treasury management, and on-chain reputation via ERC-8004.

Base URL: https://sea-lion-app-aurxb.ondigitalocean.app (or your deployment URL)

Port: 3000 (internal)

Authentication: x402 payment headers for protected endpoints. Health and stats are public.

💰 x402 Payment Protocol

Eidolon implements the x402 protocol for monetizing API endpoints. Clients pay per-call using on-chain settlement (ERC-402).

How it works:

1. Request protected endpoint Include header: X-Client-ID: your-client-id
2. Receive 402 Payment Required Response headers: X-402-Payment-Required, X-402-Payment-Address, X-402-Payment-Description
3. Pay on-chain Send specified amount (USDC) to the payment address. Include memo if required.
4. Retry request After payment is detected, endpoint returns data. Alternatively, call /webhook/credit to credit client manually.

Demo Mode

When DEMO_MODE=true in .env, the agent auto-creates a demo client with credit balance. Use X-Client-ID: demo to access paid endpoints without real on-chain payment.

x402 Protected Endpoints

GET /signals/price/:token

Get trading signal for a token. Requires payment (default $0.50).

Headers: X-Client-ID: <your-client-id>

GET /signals/price/ETH HTTP/1.1
X-Client-ID: demo

Response (200 OK):
{
  "success": true,
  "data": {

}

POST /copilot/chat

Chat with the agent's copilot (LLM). Requires payment (default $0.10 per message).

Body: { "message": "your question" }

POST /copilot/chat HTTP/1.1
X-Client-ID: demo
Content-Type: application/json

{"message": "What's the current market sentiment?"}

Response (200 OK):

GET /events

Server-Sent Events (SSE) stream of live logs. No authentication required. Used by dashboard for real-time updates.

Format: data: {"level":"info","message":"..."}\n\n

🔢 Trust-Based Pricing

Eidolon adjusts prices based on the client's ERC-8004 trust score (0-1000). Higher trust = lower prices (up to 20% discount). Lower trust = higher prices (up to 100% surcharge).

finalPrice = basePrice * (1 - (trustScore / 1000) * 0.2)
// Range: 0.8 * basePrice (trust=1000) to 2.0 * basePrice (trust=0)

Trust score is set via /admin/trust-score or automatically updated by the ReputationManager from on-chain reputation registries.

🏗️ Architecture

  • X402Server: Express server handling HTTP, x402, SSE, static assets
  • CreditLedger: Persistent ledger (JSON file) tracking client balances and debts
  • BankrClient: Wrapper for Bankr API (balances, health, on-chain data)
  • ReputationManager: ERC-8004 identity and reputation integration (optional)
  • Orchestrator: Autonomous loop (health checks, trust score sync, disabled trading)
